API Reference¶

Simulation Class¶

class amr_sim.Simulation(screen_dim, clock_rate)¶

A class for handing the displaying and interaction with the simulation environment.

Parameters

  • screen_dim (tuple of int) – the screen dimension in pixels (x, y)

  • clock_rate (int) – The rate in Hz to update the simulation

arrow_keys()¶

Function which returns the current keys which are being pressed following this format: hor = -1 -> down, hor = 1 -> up, vert = -1 -> left, vert = 1 -> right

Returns

(hor, vert), two ints depicting which arrow is currently pressed

Return type

int, int

background_set(lights)¶

Sets the background based off of a list of Light objects

Parameters

lights (List of Light) – A list of lights to overlay onto the background

change_sprite(sprite)¶

A function to to return the next sprite in the simulation, if only one sprite it returns that one and if no sprites returns None

Parameters

sprite (Sprite) – The current sprite selected

Returns

The next sprite in the list

Return type

Sprite

clock = None¶

pygame.clock: pygame clock of the simulation

draw_dot(colour, center, radius)¶

Draws a static dot on the screen but with no collision detection

Parameters

  • colour (tuple of int) – (r,g,b) The colour of the dot using rgb values ints from 0 to 255

  • center (tuple of floats or ints) – A (x, y) tuple with the coordinated of the center of the dot

  • radius (int) – The radius of the dot

draw_line(colour, start, direction, length, width)¶

Draws a static line on the screen

Parameters

  • colour (tuple of int) – (r,g,b) The colour of the line using rgb values ints from 0 to 255

  • start (tuple of int) – (x, y), The co-ordinates of the middle of the line at one end

  • direction (char) – The char ‘x’ or ‘y’ depicting the direction the line goes

  • length (int) – The length of the line in pixels

  • width (int) – The width of the line in pixels

draw_rect(colour, top_left, width, height, line_thickness)¶

Draws a static rectangle object on the screen

Parameters

  • colour (tuple of int) – (r,g,b) The colour of the line using rgb values ints from 0 to 255

  • top_left (tuple of int) – (x, y), The co-ordinates of the top left of the rectangle

:param width:The width of the rectangle from outside to outside :type width: int

Parameters

  • height (int) – The height of the rectangle from outside to outside

  • line_thickness (int) – The thickness of the rectangle boarder, if 0 the rectangle will be filled

events_detection()¶

Uses the event detection from pygame to set the arrow key variables which allow for multi key handling on the arrows. This is required if arrow_keys() function is to be used. It returns the dictionary of keys which event detection have been activated using the self.keys variable

Returns

A dict with which keys in the self.keys dict have changed, 0 = no change, 1 = key pressed, -1 = key released

Return type

dict

keys = None¶

dict: keys to check for changes, add custom keys from pygame for them to be returned in event_detection dictionary

point_collision(point)¶

Detects if a point collides with any objects or sprites

Parameters

point (pygame.math.Vector2) – A point to test

Returns

A boolean if a collision has a occurred, true for occurred, false for not

Return type

bool

running = None¶

bool: Running variable to control main loop of program, becomes False when quit event received

update_display(flip=True)¶

Updates the screen with the current position of all the sprites and static objects placed onto the screen. If the flip value is False then the displaying of this update is not performed so other objects can be placed on the screen before the update.

Parameters

flip (bool) – A bool to depict if the screen is updated or left to be done externally

value_func(point)¶

Gets the value of the combined light functions for a given point

Parameters

point (pygame.math.Vector2) – The point to get the value of all the lights from

Returns

the sum of all the light values at the given point

Return type

float

wait()¶

This function delays to keep the main loop under the set frame rate. It also saves the tick variable in the clock which allows for velocities to be calculated correctly, so must be placed at the end of the control loop when using this library

Position Class¶

class amr_sim.Position(x, y, width, height, angle=0.0)¶

This classed is imported into Sprite classes and has been constructed to allow for easier passing of coordinates + angles. It also includes functions that are used when dealing with frames of references

Parameters

  • angle (int or float) – The angle of the object in degrees, positive x axis is 0 and CCW is positive
